# References — what's in each file

Read on demand based on what the user is working on. Don't preload all of them.

- **`facilitation-methods.md`** — Step-by-step instructions for 20+ methods. Canonical home for Open Space Technology, the Minimum Viable Community (MVC) 9-question tool, and workshop design for hands-on learning (build sessions, pair/mob programming, show & tell).

- **`community-frameworks.md`** — Community design frameworks for ongoing groups. Full treatments of Community Canvas (17 themes), Community Weaving (5 patterns), and Wenger Communities of Practice. Condensed treatments of Tacit Maturity Model and JRC Playbook. Brief notes on CMX SPACES, Charles Vogl, FeverBee, Get Together.

- **`hackathons-and-builds.md`** — Event-format design for hackathons, unconferences, and design sprints. Unconferences as an *event format* applying Open Space live here (the underlying method lives in facilitation-methods).

- **`social-learning-evidence.md`** — Research backbone. Foundational theories (Bandura, Vygotsky, Lave & Wenger), peer learning effect sizes, cohort completion data, psychological safety research (Edmondson, Project Aristotle), group size studies. Use this when the user needs citations or wants to convince a stakeholder.

- **`facilitator-pain-points.md`** — Common mistakes, advanced challenges, non-obvious wisdom. The "what goes wrong" lens. Read it when the user is troubleshooting or preparing for a difficult gathering.
